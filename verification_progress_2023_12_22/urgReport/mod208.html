318                         ---------------------------------------------------------------------------
319                         -- On RTL, can_tx is 'U' at time zero, and it gets defined value when
320                         -- res_n is asserted. Thus 'U' -> 1 event occurs in non-zero time.
321                         -- On Xilinx gate level sims, having res_n = 'U' first few nanoseconds of
322                         -- simulation does cause output of flop in reset synchronizer to be '0',
323                         -- not 'U'. Thus synchronized reset is '0' from time 0, and there is no
324                         -- event on it when res_n input gets asserted non-'U' value! This causes
325                         -- can_tx to be set to '1' from time 0 of simulation. As consequence of
326                         -- this, signal delayer will ignore the first event on can_tx in time 0,
327                         -- and will keep its output at 0!
328                         ---------------------------------------------------------------------------
